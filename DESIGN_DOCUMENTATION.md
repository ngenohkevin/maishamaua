## Brand Philosophy

### Core Message
> **"Give them flowers while they are still here"**

Maisha Maua is built on the philosophy of **intentional appreciation** and the **celebration of life**. The brand focuses on honoring individuals in the present moment.

### Key Brand Values
- **Intentional Appreciation** - Celebrating people while they're here
- **Customization** - Bouquets designed to reflect personality and preferences
- **Meaningful Gifting** - Flowers paired with complementary gifts
- **Life Celebration** - For appreciation, milestones, and everyday recognition

### Important Non-Negotiable
**Maisha Maua does NOT provide flowers for funerals or burials.**
This is a core principle that reflects the brand's focus on celebrating life in the present.

### Target Occasions & Relationships
**Occasions:**
- Friend appreciation
- Family celebrations
- Colleague recognition
- Romantic gestures
- Milestones & achievements
- "Just because" moments
- Birthdays & graduations
- Promotions & successes

**Relationships:**
- Friends
- Family members
- Colleagues
- Loved ones

## Design Direction

### Chosen Design Elements
Sophie selected elements from **Design 2 (Fresh & Modern)** and **Design 4 (Classic Romantic)**, with emphasis on the **fonts**.

#### From Design 2 (Fresh & Modern)
- Clean, modern layout
- Sans-serif typography for body text
- Sage green accent colors (`#4A5D48`)
- Rounded corners on cards
- Light, airy feel
- WhatsApp-green CTAs (`#25D366`)

#### From Design 4 (Classic Romantic)
- **Playfair Display** serif font for headings
- Soft, warm color palette
- Elegant italic styling
- Romantic, sophisticated feel
- Blush/dusty rose accents (`#C4A59E`)

### Final Color Palette

| Color | Hex Code | Usage |
|-------|----------|-------|
| Sage Green | `#4A5D48` | Buttons, accents, badges |

## Image Optimization Guidelines

### Current State
- All images are JPEG format
- Average size: ~250 KB per image
- No WebP conversion yet

### Recommended Optimizations

| Optimization | Benefit | Priority |
|--------------|---------|----------|
| Convert to WebP | 25-35% smaller file sizes | High |
| Implement lazy loading | Faster initial page load | Medium |
| Add responsive sizes | Serve appropriate size per device | Medium |
| Use Next.js Image component | Automatic optimization | ✅ Done |

### Next.js Image Component Usage
```tsx
<Image
  src="/images/hero-bouquet.jpeg"
  alt="Beautiful flower bouquet"
  fill
  className="object-cover"
  priority  // For above-the-fold images
/>
```

### Future WebP Migration
```bash
# Convert all JPEGs to WebP (future task)
for file in *.jpeg; do
  cwebp -q 80 "$file" -o "${file%.jpeg}.webp"
done
```

```typescript
export default function robots(): MetadataRoute.Robots {
  return {
    rules: { userAgent: "*", allow: "/" },
    sitemap: "https://maishamaua.iopulse.cloud/sitemap.xml",
  }
}
```

### sitemap.ts Configuration
```typescript
export default function sitemap(): MetadataRoute.Sitemap {
  return [
    { url: baseUrl, changeFrequency: "weekly", priority: 1 },

    { url: `${baseUrl}/#about`, changeFrequency: "monthly", priority: 0.5 },
    { url: `${baseUrl}/#contact`, changeFrequency: "monthly", priority: 0.5 },

}
```

## User Flow

### Primary Flow: Order via WhatsApp
1. User lands on homepage
2. Sees hero with brand message
3. Scrolls to view products with prices
4. Clicks "Order" on any product
5. Opens WhatsApp Business chat
6. Completes order conversation with Sophie

### Secondary Flow: Browse & Inquire
1. User browses product catalog
2. Views custom bouquet options
3. Clicks "Inquire About Custom Orders"
